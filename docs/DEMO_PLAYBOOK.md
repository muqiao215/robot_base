# 演示操作手册（玻璃反射环境下的稳妥路径）

> 适用于：走廊 / 门口 / 玻璃幕墙区域的真实导航演示。
> 核心约束：**玻璃反射导致激光 ±5cm 漂移，无法靠配置根治，只能靠流程绕开。**

---

## 演示前 30 秒必做清单

- [ ] 1. **2D Pose Estimate 精确定位**：箭头位置放到机器人真实位置，**反复微调朝向**直到激光点云"咬"住墙。位置差 20cm 没事，朝向差 10° 点云全错开。
- [ ] 2. **确认 footprint 是 +0.05m 加宽版**：`ros2 param get /local_costmap/local_costmap footprint` 应是 `[[0.30, 0.31], ...]`。
- [ ] 3. **确认 inflation 不夸张**：global inflation ≤ 0.35、local ≤ 0.30（已配置）。
- [ ] 4. **点之间距 ≤ 1.5m**，避免远距离一键导航把车带到视野外的不可控位置。
- [ ] 5. **每个目标点放在可见的中线上**，不要压在墙/家具边缘。

---

## waypoint 点位记录（按场景填）

格式：`index | x (map) | y (map) | yaw_deg | 备注`

```yaml
# 示例：演示点列从起点到走廊尽头
waypoints:
  - idx: 1
    x: 0.0
    y: 0.0
    yaw_deg: 0.0
    note: 起点/演示起点
  - idx: 2
    x: 1.0
    y: 0.0
    yaw_deg: 0.0
    note: 进门前 1m，平地
  - idx: 3
    x: 1.5
    y: 0.0
    yaw_deg: 0.0
    note: 门口前 0.5m，靠近门口
  - idx: 4
    x: 2.2
    y: 0.0
    yaw_deg: 0.0
    note: 门洞中线（玻璃反射最强位置）
  - idx: 5
    x: 2.9
    y: 0.0
    yaw_deg: 0.0
    note: 出门后 0.5m，已脱离玻璃区
```

**记录方法**：每到一个想定的 waypoint，在 RViz **用 2D Pose Estimate（不是 Goal Pose）**点击该点 → 从日志读 `amcl` 的 `Setting pose` 行就是当前 map 坐标。或者直接 `ros2 topic echo /amcl_pose --once`。

## 演示中的应急

- **车突然停下/转圈**：先看 RViz 激光点云是否仍在贴墙；如果"糊开"了，说明玻璃反射又出来，**就地 2D Pose Estimate** 重定位，然后发下一个 waypoint。
- **车偏离中线**：立刻 **2D Pose Estimate** 把它拉回正确轨迹，而不是等 Nav2 自己纠正（它可能会卡住）。
- **完全失控**（撞向玻璃、车头朝错向）：停车 `ros2 topic pub --once /cmd_vel geometry_msgs/msg/Twist "{}"`，物理把车推回中线，重新走流程。

---

## 不要做的事（演示中）

- ❌ 不要从远处一键发最终目标
- ❌ 不要在玻璃区附近设置 `tolerance` < 0.3m 的点（车会反复"未到达"重规划）
- ❌ 不要相信 waypoint-follower 自带的 auto-recovery——它在玻璃区会触发更多旋转，制造更多反射漂移

---

## 中间点 tolerance 放宽建议

如果用 `navigate_through_poses`（waypoint 序列），给中间点 `goal_checker.xy_goal_tolerance: 0.4`（已在配置里），这样中间点不严格到达也过，避免"差一点停住"的死循环。

---

## 流程 vs 配置：为什么要这样

| 解决思路 | 是否可行 | 原因 |
|---|---|---|
| 改激光参数过滤玻璃反射 | 不可行 | 玻璃 + 墙面距离 < 1m 时，反射光斑与真实扫描混杂，无法分离 |
| 加 footprint 让规划更保守 | **部分可行**（已用 +0.05m）| 给激光漂移留余量，但加太多会把通道堵死 |
| inflation 加大让车离墙远 | **部分可行**（保持当前 0.30/0.35 不动）| 太大反而把门洞堵死 |
| 端到端远距离一键导航 | **不可行** | 远距离意味着至少有一段穿过玻璃区，必然撞幻影墙 |
| **点位一段一段发，演示前重定位** | **可行（流程兜底）** | 把"穿越玻璃区"分解成"短距离 + 已知好轨迹 + 视觉确认"，每步可中断可纠正 |

`footprint +0.05m` + `inflation ≤ 0.35` + **点列演示 + 每步重定位**，三者配合，就是这套玻璃环境下的最优解。
